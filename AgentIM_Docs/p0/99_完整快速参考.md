## 一句话架构

```
前端 ← HTTP/WS → Gateway → Auth + Business(im) → PG16 / Redis / RocketMQ / ES / MinIO
```

## P0 核心表速查

| 表名 | 一句话业务含义 | 关键字段 |
|------|--------------|----------|
| `im_user` | 用户资料 + @handle 用户名 + 用户级数据策略预留 | username, nickname, avatar, bio, data_scope, permission_tags, access_policy |
| `im_user_contact` | 联系人关系 | user_id, contact_user_id, remark |

## 核心 API 速查

```http
# User
GET    /im/users/profile                          # 当前用户资料
PUT    /im/users/profile                          # 更新资料
GET    /im/users/search?q=xxx                     # 搜索用户
GET    /im/users/{userId}/access-policy           # 查询用户级数据策略预留
PUT    /im/users/{userId}/access-policy           # 更新用户级数据策略预留

# Contact
GET    /im/contacts                               # 联系人列表
POST   /im/contacts                               # 添加联系人

# Chat
GET    /im/chats                                  # 聊天列表
POST   /im/chats                                  # 创建群组/频道/私聊
GET    /im/chats/{id}                             # 聊天详情
PUT    /im/chats/{id}                             # 更新设置
DELETE /im/chats/{id}                             # 删除/退出
POST   /im/chats/{id}/archive                     # 归档
POST   /im/chats/{id}/members                     # 邀请成员
DELETE /im/chats/{id}/members/{userId}            # 移除成员

# Message（核心）
GET    /im/chats/{id}/messages?beforeSeq={seq}    # 消息列表（游标分页）
POST   /im/chats/{id}/messages                    # 发送消息
PUT    /im/messages/{id}                          # 编辑消息
DELETE /im/messages/{id}                          # 对所有人删除
POST   /im/messages/{id}/hide                     # 仅自己隐藏

# Reaction
POST   /im/messages/{id}/reactions                # 加 emoji

# Read
POST   /im/messages/{id}/read                     # 标记已读
GET    /im/messages/{id}/read-status              # 消息已读状态（飞书式）
GET    /im/chats/{id}/unread-count                # 未读数

# Pin
POST   /im/messages/{id}/pin                      # 置顶
DELETE /im/messages/{id}/pin                      # 取消置顶
GET    /im/chats/{id}/pinned                      # 置顶列表

# Resource
POST   /im/resources/upload                       # 上传文件
GET    /im/resources/{id}/download                # 下载文件

# Search
GET    /im/search?q=xxx                           # 全局搜索消息
GET    /im/search?q=xxx&chatId={id}               # 聊天内搜索

# WebSocket
ws://host/ws/im?token=xxx                         # 连接
```

## 常用索引（PostgreSQL）

```sql
-- 消息列表
CREATE INDEX idx_message_chat_seq ON im_message (chat_id, seq);

-- 消息全文搜索（数据库兜底，主力用 ES）
CREATE INDEX idx_message_fts ON im_message
  USING GIN (to_tsvector('simple', content));

-- 幂等唯一（部分索引）
CREATE UNIQUE INDEX uk_message_idempotent ON im_message (chat_id, idempotent_key)
  WHERE idempotent_key IS NOT NULL;

-- 成员关系
CREATE UNIQUE INDEX uk_chat_user ON im_chat_member (chat_id, user_id);
CREATE INDEX idx_chat_member_user ON im_chat_member (user_id);

-- 已读游标
CREATE UNIQUE INDEX uk_read_state ON im_message_read_state (user_id, chat_id);

-- 反应
CREATE UNIQUE INDEX uk_reaction ON im_message_reaction (message_id, user_id, reaction);

-- 联系人
CREATE UNIQUE INDEX uk_contact ON im_user_contact (user_id, contact_user_id);

-- 置顶
CREATE INDEX idx_pinned_chat ON im_pinned_message (chat_id, pinned_time DESC);

-- 用户隐藏消息
CREATE UNIQUE INDEX uk_user_hide ON im_user_message_hide (user_id, message_id);
```

## 关键设计约束（写代码前必读）

```
1. ❌ 消息表不要直接存文件内容（用 resource_ids 引用）
2. ❌ 不要在事务内调用外部 API（文件上传、HTTP 请求）
3. ❌ 不要跳过审计日志（消息删除、聊天创建等必须记录）
4. ❌ 私聊不能超过 2 人
5. ❌ 频道普通成员不能发消息
6. ✅ 消息列表用 seq 游标分页，不用 OFFSET
7. ✅ 已读游标只前进不后退（if new > old then update）
8. ✅ WebSocket 事件在事务提交后推送（publishAfterCommit）
9. ✅ 消息发送用 idempotentKey 防重复
10. ✅ 媒体消息必须有 resourceIds
11. ✅ 消息编辑有时间限制（默认 48 小时）
12. ✅ 对所有人删除有时间限制（默认 48 小时）
```
