# 事件、搜索、文件、审计与安全

## 概述

本章覆盖 AgentIM P0 的横切关注点——**领域事件**（所有业务变更的发布/订阅）、**搜索**（消息和用户检索）、**文件管理**（上传/下载/权限）、**审计日志**（关键操作的可追溯记录）、**安全策略**。

## 领域事件

### 统一事件格式

所有领域事件使用统一的结构：

```json
{
  "eventId": "evt_a1b2c3d4e5f6",
  "eventType": "message.created",
  "chatId": 1800000000000000002,
  "actorId": 1001,
  "payload": {},
  "occurredAt": "2026-05-13T10:00:00"
}
```

| 字段 | 说明 |
|------|------|
| `eventId` | 全局唯一事件 ID（`evt_` + UUID） |
| `eventType` | 事件类型标识 |
| `chatId` | 关联聊天（可为 null） |
| `actorId` | 操作人（可为 null，如系统事件） |
| `payload` | 事件业务数据 |
| `occurredAt` | 事件发生时间 |

### P0 事件范围

```
消息事件：
  message.created         ← 消息发送
  message.edited          ← 消息编辑
  message.deleted         ← 消息对所有人删除
  message.hidden          ← 消息仅自己隐藏
  message.pinned          ← 置顶消息
  message.unpinned        ← 取消置顶

反应事件：
  reaction.created        ← Emoji 添加
  reaction.deleted        ← Emoji 删除

已读事件：
  read_state.updated      ← 已读游标更新（触发所有成员刷新饼图/对勾）

聊天事件：
  chat.created            ← 聊天创建
  chat.updated            ← 聊天更新/成员变更
  chat.deleted            ← 聊天删除

用户事件：
  user.online             ← 用户上线
  user.offline            ← 用户离线
  user.status             ← 用户状态变化

输入事件：
  typing                  ← 正在输入
```

### RocketMQ 事件配置

```yaml
rocketmq:
  name-server: 127.0.0.1:9876
  producer:
    group: agentim-event-producer
  consumer:
    group: agentim-event-consumer

agentim:
  event:
    topic: im-event                    # 统一事件 Topic
    consumer-tag: websocket            # 消费者 Tag
```

### 事件发布机制

```java
// 在 Service 事务中注册事件
eventPublisher.publishToChat("message.created", chatId, actorId, payload);

// 内部实现：
// 1. 构建统一事件对象
// 2. 注册 TransactionSynchronization
// 3. 事务提交后 → rocketMQTemplate.convertAndSend("im-event", dto)
// 4. 各节点消费 Topic，按成员关系过滤，推送给在线用户
```

## 搜索

### 搜索架构

AgentIM 使用 **Elasticsearch** 作为核心搜索引擎，直接从 P0 开始接入。消息写入时同步索引 ES，搜索请求直接查询 ES。

```
消息发送 → 事务提交 → RocketMQ "im-event" Topic
    → EventConsumer 消费事件
    → 调用 SearchService.indexMessage(message)
    → 写入 Elasticsearch
```

### ES 索引设计

```
消息索引 (im_message)：
  {
    "id": "long",             // 消息 ID
    "chat_id": "long",        // 所属聊天
    "sender_id": "long",      // 发送者
    "sender_name": "text",    // 发送者用户名（用于搜索）
    "message_type": "keyword",
    "content": "text",        // 全文搜索字段
    "mentioned_user_ids": "long[]",
    "seq": "long",
    "status": "keyword",
    "created_at": "date"
  }
```

### 消息搜索

```json
// 全局搜索 — 查询 ES
POST /im_message/_search
{
  "query": {
    "bool": {
      "must": [
        { "match": { "content": { "query": ":keyword", "operator": "and" } } },
        { "terms": { "chat_id": [":userVisibleChatIds"] } },
        { "terms": { "status": ["normal", "edited"] } }
      ]
    }
  },
  "sort": [ { "seq": { "order": "desc" } } ],
  "size": 20
}

// 聊天内搜索
POST /im_message/_search
{
  "query": {
    "bool": {
      "must": [
        { "match": { "content": ":keyword" } },
        { "term": { "chat_id": ":chatId" } },
        { "terms": { "status": ["normal", "edited"] } }
      ]
    }
  },
  "sort": [ { "seq": { "order": "desc" } } ],
  "size": 20
}
```

### 用户搜索

用户搜索使用 PostgreSQL 的 `ILIKE` 或 ES 用户索引：

```
GET /im/users/search?q=zhang
→ SELECT * FROM im_user WHERE username ILIKE '%zhang%' OR nickname ILIKE '%zhang%'
→ 排除已删除的用户
→ 分页返回
```

### 搜索配置

```yaml
spring:
  elasticsearch:
    uris: http://elasticsearch:9200

agentim:
  search:
    index-prefix: agentim_
    message-index: agentim_im_message
    index-refresh-seconds: 1      # ES 索引刷新间隔
    highlight-tag: <em>           # 高亮标签
```

## 文件管理

### 文件上传下载流程

```
上传阶段（独立于消息）：
  前端选择文件
    → POST /im/resources/upload (multipart)
    → 后端校验文件类型和大小
    → 上传到 OSS/MinIO/S3
    → 图片自动生成缩略图
    → 创建 im_resource 记录
    → 返回 resourceId
    └→ 前端拿到 resourceId

发消息阶段：
  用户在消息中引用已上传的文件
    → POST /im/chats/{chatId}/messages
    → body: { messageType: "image", resourceIds: [resourceId] }
    → 后端关联 resource 到 chat 和 message

下载阶段：
  用户点击文件
    → GET /im/resources/{resourceId}/download
    → 后端鉴权（聊天成员关系）
    → 生成临时下载 URL 或重定向到 OSS
    └→ 前端下载文件
```

### 文件权限模型

```
im_resource.access_level 字段控制文件访问范围：

"private" → 仅上传者可访问
"chat"    → 聊天所有成员可访问（默认）
"public"  → 任何人可访问（仅限头像等公共资源）
```

### 文件存储配置

```yaml
agentim:
  file:
    provider: minio          # oss / minio / s3
    bucket: agentim-files
    access-key: xxx
    secret-key: xxx
    endpoint: http://minio:9000
    max-size: 209715200       # 单文件最大 200MB
    allowed-types: >-
      image/jpeg,image/png,image/gif,
      video/mp4,
      audio/ogg,audio/mpeg,
      application/pdf,
      text/plain
    thumbnail: true           # 自动生成缩略图
    download-url-expire: 3600  # 下载链接有效期（秒）
```

## 审计日志

### 审计原则

- **只追加**（append-only），不会修改或删除
- **不可抵赖**，记录操作人、时间、内容
- **结构化**，便于查询和分析

### 审计日志表

```sql
im_audit_log
├── id               BIGINT          -- 主键
├── chat_id          BIGINT          -- 聊天 ID
├── resource_type    VARCHAR(60)     -- 资源类型：message / chat / user
├── resource_id      BIGINT          -- 资源 ID
├── action           VARCHAR(100)    -- 操作：DELETE_MESSAGE_ALL、CREATE_CHAT 等
├── actor_id         BIGINT          -- 操作人
├── summary          VARCHAR(1000)   -- 人类可读的摘要
├── ipaddr           VARCHAR(128)    -- 客户端 IP
├── user_agent       VARCHAR(500)    -- 客户端 User-Agent
├── payload          JSONB           -- 详细载荷
└── occurred_time    TIMESTAMP(6)    -- 发生时间
```

### P0 必须审计的操作

| 操作 | 审计内容 |
|------|----------|
| 消息对所有人删除 | 操作人、消息 ID、聊天 ID、内容摘要 |
| 消息编辑 | 操作人、消息 ID、修改前后内容摘要 |
| 创建/删除聊天 | 操作人、聊天名称、类型、成员列表 |
| 添加/移除成员 | 操作人、被操作人、角色 |
| 置顶/取消置顶 | 操作人、消息 ID |
| 上传文件 | 操作人、文件名、大小 |

## 安全策略

### 安全防护全景

```
┌──────────────────────────────────────────────────────┐
│                    安全防护全景图                       │
├──────────────────────────────────────────────────────┤
│                                                      │
│  [网络]     TLS 传输加密 + WAF / 反向代理              │
│                                                      │
│  [认证]     Sa-Token 登录认证 + Token 管理             │
│                                                      │
│  [权限]     @SaCheckPermission + 聊天成员关系          │
│                                                      │
│  [文件]     上传类型白名单 + 大小限制 + 下载鉴权        │
│                                                      │
│  [内容]     敏感词过滤 + 垃圾消息防护 + 举报机制        │
│                                                      │
│  [消息]     XSS 防护 + 内容长度限制                    │
│                                                      │
│  [审计]     关键操作记录 + 不可篡改 + 可追溯            │
│                                                      │
│  [数据]     SQL 注入防护 + 字段加密 + 敏感信息脱敏     │
│                                                      │
│  [合规]     消息留存 + 日志保留 + 审计不可删除         │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 具体安全规则

```
1. 认证安全：
   - 所有接口通过 Sa-Token 校验登录态
   - Token 有过期时间，过期后需重新登录
   - 密码使用加密存储

2. 文件安全：
   - 文件上传必须登录
   - 文件类型白名单（防止恶意文件上传）
   - 单文件大小限制（默认 200MB）
   - 下载必须二次鉴权（不能直接通过 URL 访问 OSS）
   - 下载链接有过期时间（默认 1 小时）

3. 消息安全：
   - 消息内容在前端渲染时做 XSS 转义
   - 消息内容有长度限制（文本默认 4096 字符）
   - 文件消息必须引用合法 resourceId

4. 聊天安全：
   - 非成员无法查看聊天消息
   - 非成员无法发送消息
   - 频道限制了发言权限
   - 私聊只有双方可见

5. 内容安全：
   - 消息发送前检查敏感词（可配置词库）
   - 命中敏感词：记录审计日志 + 根据配置决定是否阻止发送
   - 单用户发送频率限制（默认每分钟 30 条）
   - 相同内容防刷（短时间内相同内容合并为一条）

6. 数据加密：
   - 传输层：全链路 TLS 1.3
   - 存储层：PostgreSQL 可配置 TDE（透明数据加密）
   - 应用层：敏感配置通过 Nacos AES 加密存储
   - Token/Secret：加密存储于数据库（AES-256-CBC）
   - 消息内容：P0 不加密（因为需要 ES 全文检索），P1 可选端到端加密

7. 合规要求：
   - 消息删除操作为逻辑删除，原始内容保留（满足数据留存法规）
   - 审计日志只追加不删除，至少保留 180 天（可配置）
   - 用户注销账号时，消息记录可选择匿名化或保留
   - 所有审计日志包含 IP 地址，可用于追溯

8. 幂等与重试：
   - 消息发送 → idempotentKey + 唯一索引
   - 已读更新 → 只前进不后退
   - 反应添加 → 唯一约束幂等
   - 隐藏消息 → ON CONFLICT DO NOTHING
```
