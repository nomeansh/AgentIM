# 后端模块结构、枚举与权限

## 整体模块架构

AgentIM 基于 RuoYi-Cloud-Plus 微服务架构改造，但专注于 IM 核心场景——保留了网关、认证、资源、WebSocket 等核心底座。

```
AgentIM-Business/          ← 主要业务服务（P0 IM 全部代码）
AgentIM-gateway/           ← 网关（路由 /auth/** /im/** /ws/**）
AgentIM-auth/              ← 认证服务（登录、注册、令牌）
AgentIM-modules/
  AgentIM-resource/        ← 资源服务（OSS 文件上传）
AgentIM-api/               ← Dubbo 远程接口契约
  AgentIM-api-auth/        ← 认证契约
  AgentIM-api-resource/    ← 资源远程接口
AgentIM-common/            ← 公共能力
  AgentIM-common-websocket/
  AgentIM-common-sse/
  AgentIM-common-rocketmq/    ← RocketMQ 事件广播（替代 Redis Pub/Sub）
  AgentIM-common-redis/
  AgentIM-common-elasticsearch/
```

## 权限体系

### 权限标识定义

AgentIM 的权限模型遵循 Sa-Token 的 `@SaCheckPermission` 注解体系。

| 权限标识 | 接口 | 说明 |
|----------|------|------|
| `im:user:profile` | GET /im/users/profile | 查看/编辑个人资料 |
| `im:contact:list` | GET /im/contacts | 查看联系人列表 |
| `im:contact:add` | POST /im/contacts | 添加联系人 |
| `im:contact:remove` | DELETE /im/contacts | 删除联系人 |
| `im:chat:list` | GET /im/chats | 查看聊天列表 |
| `im:chat:create` | POST /im/chats | 创建聊天（群组/频道） |
| `im:chat:update` | PUT /im/chats/{id} | 更新聊天设置 |
| `im:chat:delete` | DELETE /im/chats/{id} | 删除聊天 |
| `im:chat:member` | POST/DELETE /im/chats/{id}/members | 管理成员 |
| `im:user:access-policy` | GET/PUT /im/users/{userId}/access-policy | 查询/更新用户级数据访问策略预留字段 |
| `im:message:send` | POST /im/chats/{id}/messages | 发送消息 |
| `im:message:read` | GET /im/chats/{id}/messages | 读取消息 |
| `im:message:edit` | PUT /im/messages/{id} | 编辑消息 |
| `im:message:delete` | DELETE /im/messages/{id} | 删除消息 |
| `im:message:pin` | POST /im/messages/{id}/pin | 置顶消息 |
| `im:reaction:add` | POST /im/messages/{id}/reactions | 添加反应 |
| `im:resource:upload` | POST /im/resources/upload | 上传文件 |

### 数据权限（聊天级别）

除了 Sa-Token 的权限标识校验，还通过成员关系做数据级权限：

```
私聊：
  - 仅双方参与者可见
  - 双方都可发消息、标记已读、删除消息

群组：
  - 群组成员可见
  - owner/admin → 可管理成员、修改群设置、删除任何人的消息
  - 普通成员 → 可发消息、删自己的消息

频道：
  - 订阅者（subscriber）→ 可见频道消息、不可发言
  - 管理员 → 可发消息、管理频道

保存的消息：
  - 仅自己可见
```

### 用户级策略预留

聊天之外的业务数据可能需要对部分用户关闭或限制访问，例如内部资料、运营数据、实验功能、未来 Workspace 资源等。P0 在 `im_user` 预留用户级策略字段：

- `data_scope`：用户默认数据范围，支持 `standard`、`restricted`、`trusted`、`blocked`
- `permission_tags`：用户权限标签数组，用于后续 ABAC/标签匹配
- `access_policy`：用户访问策略对象，可承载 allowTags、denyTags、resourceScopes、denyReason 等扩展配置

这些字段不替代 `im_chat_member`。聊天消息、群组、频道、资源下载、已读、置顶等 P0 IM 数据仍以成员关系和角色做二次授权。
