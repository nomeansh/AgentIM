# 阅读说明

## 这份文档是什么？

这份文档是 **AgentIM** 项目的完整设计规格说明书。AgentIM 是一个面向个人与团队的 **开源即时通讯平台**——你可以把它理解为一个可私有部署的 Telegram，具备 **Telegram 的云消息体验**、**丰富的消息类型**、**频道广播能力**以及**多设备同步**。

> **产品参考对标**：
> - **Telegram** — 云消息、多设备、频道、富媒体
> - **WhatsApp** — 社区、频道、状态、强移动端体验
> - **Mattermost** — 开源、私有部署、团队协作
> - **Slack** — 频道化沟通、集成生态
> - **X** — 公开内容、Spaces、列表、创作者分发
> - **Signal** — 隐私、用户名、阅后即焚、端到端加密
> - **Discord** — 社区沟通、语音

### 一句话说清 AgentIM 是什么

```
AgentIM = 云消息（Telegram 式消息同步 + 富媒体）
       + 聊天（私聊 / 群组 / 频道）
       + 多设备（云端同步，多端登录）
       + 频道广播（一对多订阅）
```

### 三阶段演进路线

AgentIM 按三个阶段交付：

| 阶段 | 聚焦 | 用户故事 |
|------|------|----------|
| **P0 — 核心 IM** | 像 Telegram 一样聊天的云消息平台 | "我可以注册账号、设置用户名、加好友、在私聊/群组/频道中收发消息和文件，所有消息在多个设备间同步" |
| **P1 — 增强协作与社区** | 工作区、超级群、话题、状态、语音房、隐私增强、Agent 协作 | "我可以把团队或社群放进 AgentIM，用工作区和社区管理成员与讨论，让 Agent 在频道里协助完成任务" |
| **P2 — 开放生态与增长** | Bot API、Mini Apps、工作流、公开频道、DevOps、企业治理 | "我可以把外部系统、机器人、小程序、公开频道和企业治理接入 AgentIM，让沟通、自动化和运营闭环" |

## 核心设计哲学

1. **云消息是核心** — 所有消息存储在云端，用户可以在任意设备上访问完整的历史记录。消息同步是第一优先级，不是事后补充。

2. **先做简单的事** — P0 只做核心 IM，不做 Agent、审批、DevOps。消息底座稳定了，才有上层建筑。

3. **富媒体是标配** — 不只是文本，图片、视频、语音、文件、投票都是消息的一等公民。

4. **消息操作必须完整** — 发送、编辑、删除（对所有人/仅自己）、回复、转发、置顶、搜索，每条消息都有完整的生命周期。

5. **多设备同步** — 用户可以在手机、桌面、Web 等设备同时登录，消息状态（已读、删除、置顶）在所有设备间一致。

6. **跨系统动作全程审计** — 所有关键操作（删除消息、创建频道、成员变更）都写入审计日志。

## 核心术语表

| 术语 | 英文 | 说明 |
|------|------|------|
| 用户 | User | 系统参与者，拥有唯一用户名（@handle）和个人资料 |
| 私聊 | Private Chat | 一对一私密会话 |
| 群组 | Group | 多人聊天会话，所有成员可发言 |
| 频道 | Channel | 一对多广播，订阅者只能读不能写 |
| 消息 | Message | 沟通的基本单位，支持文本/图片/文件/语音/视频/投票等 |
| 已读游标 | Read State | 追踪用户在聊天中已读到的位置 |
| 置顶 | Pinned Message | 固定在聊天顶部的消息 |
| 引用回复 | Reply | 对某条消息的针对性回复 |
| 转发 | Forward | 将消息从当前聊天转发到另一个聊天 |
| 联系人 | Contact | 用户之间的好友关系 |
| 保存的消息 | Saved Messages | 与自己的私聊，用于云端存储 |
