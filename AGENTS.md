## 当前阶段：P0 核心 IM

P0 目标：构建一个类似 Telegram 的开源即时通讯平台。核心能力包括：

- 用户系统（@handle 用户名、在线状态）
- 联系人管理
- 聊天会话（私聊 / 群组 / 频道 / 保存的消息）
- 富媒体消息（文本 / 图片 / 文件 / 语音 / 视频 / 投票）
- 消息操作（编辑 / 删除 / 引用回复 / 转发 / 置顶）
- Emoji 反应
- 已读游标 + 未读数
- WebSocket 实时推送
- 多设备同步
- 消息搜索
- 文件上传（OSS/MinIO/S3）
- 审计日志

## 模块结构

```
AgentIM-Business/src/main/java/com/AgentIM/business/
  im/                          ← P0：IM 核心业务
    user/                      ← 用户资料、在线状态、联系人
    chat/                      ← 聊天会话 CRUD
    member/                    ← 聊天成员管理
    message/                   ← 消息（编辑/删除/回复/转发）
    reaction/                  ← Emoji 反应
    readstate/                 ← 已读游标
    resource/                  ← 文件资源
    poll/                      ← 投票
    search/                    ← 搜索
    pin/                       ← 置顶
    saved/                     ← 保存的消息
    event/                     ← 领域事件 → WebSocket
    permission/                ← 权限校验
```

## 编码要求

- 所有正文注释、接口说明、提交说明优先使用中文。
- 遵循 AgentIM 当前模块命名与 RuoYi-Cloud-Plus 既有分层风格（Controller → Service → Mapper）。
- Controller 返回统一 `R<T>`；分页接口使用 `TableDataInfo`。
- 不要绕过 Sa-Token 权限体系。
- 新表必须带通用审计字段（create_by/create_time/update_by/update_time/del_flag）。
- 权限标识使用 `im:*` 前缀。
- **事务边界**：事务内只做数据库操作，外部文件/网络调用在事务外通过领域事件触发。

## 禁止事项

- 禁止在消息表中直接存放文件内容。
- 禁止跳过审计日志。
- 禁止在事务内进行耗时的外部调用。
- 禁止私聊超过 2 人。
- 禁止频道普通成员发言。
- 禁止缺失幂等处理（消息发送必须用 idempotentKey 防重复）。
